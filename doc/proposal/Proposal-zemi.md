# 06-09のゼミ

## 概要
本研究では、時系列表現を「系列全体に共通する情報」と「局所的に変化する情報」に分けて扱う既存研究を出発点とし、小売需要データに適用する際の課題を整理する。

対象とする FreshRetailNet-50K では、売上そのものを直接予測するだけでは、単純な基準値が非常に強く、モデルが何を学習しているのかを解釈しにくい。そこで本研究では、売上 $y$ から基準値 $b$ を引いた残差

```math
r = y - b
```

を学習対象にする。さらに、この残差を

```math
r_{i,d,h}
=
g_i
+ a_{i,d}
+ c_{i,h}
+ u_{i,d,h}
+ \varepsilon_{i,d,h}
```

のように、店舗・商品系列、日、時間帯、日と時間帯の組み合わせに分けて考える。

この方針により、単に予測精度を上げるだけでなく、「どの時間帯に基準値から外れやすいのか」「どの基準値を使うと残差に構造が残るのか」を説明できるようにする。

# 1. decoupling local and global representations の論文について

## 1.1 元論文の問題意識

元論文である *Decoupling Local and Global Representations of Time Series* は、時系列データから得られる潜在表現を、異なる性質を持つ複数の表現に分けることを目的としている。

基本的な考え方は、時系列には大きく分けて次の 2 種類の情報が含まれる、というものである。

| 表現 | 意味 | 小売データでの例 |
|---|---|---|
| global | 系列全体を通じて比較的変わりにくい性質 | ある店舗・商品が普段どの程度売れるか |
| local | 時点や短い区間で変化する性質 | ある日、ある時間帯だけ売上が変化すること |

この枠組みを小売需要に当てはめると、global は「店舗・商品ごとの基本的な売れやすさ」、local は「日や時間帯による一時的な変化」と考えられる。

## 1.1.1 周辺研究の中での位置づけ

時系列の表現分離には、大きく 4 つの流れがある。

| 流れ | 代表例 | 本研究との関係 |
|---|---|---|
| static/dynamic または global/local の潜在分離 | FHVAE, DSVAE, C-DSVAE, Decoupling Local and Global Representations | 元論文の直接的な系譜 |
| self-supervised / contrastive 表現学習 | TNC, TS2Vec, TS-TCC, TF-C | 良い時系列表現を作るが、成分の意味は暗黙的 |
| global-local forecasting / decomposition | Deep Factors, DeepGLO, CoST, Autoformer, FEDformer, TimeMixer | 予測で大域構造と局所補正、trend/season 分解が有効であることを示す |
| local pattern / anomaly evaluation | Shapelets, INSHAPE, Anomaly Transformer, TranAD | 局所パターンや high residual case を実用的に評価する流れ |

これらの研究は、時系列には複数の因子が混ざっており、それを分けることが有効であることを示している。

一方で、多くの研究は「潜在表現を分ける」または「trend/season のような粗い成分に分ける」ことが中心である。本研究では、そこから一歩進めて、強い基準値の後に残る residual correction そのものを、

```text
series / day / hour / day-hour interaction
```

という小売運用上の単位に分ける。

このため、本研究は元論文の単純な拡張というより、local/global 分離の考えを residual output decomposition へ移す研究として位置づける。

## 1.2 元論文に近い Encoder / Decoder の形

元論文に近い構成では、入力時系列 $x$ から global 表現と local 表現を作る。

```math
z_g = \mathrm{Enc}_g(x_{1:T})
```

```math
z_{l,t} = \mathrm{Enc}_l(x_{t-w:t})
```

ここで、$z_g$ は系列全体の表現、$z_{l,t}$ は時点 $t$ 周辺の局所表現である。

Decoder はこれらをまとめて受け取り、時点 $t$ の値を再構成する。

```math
\hat{x}_t
=
\mathrm{Dec}(z_g, z_{l,t})
```

学習では、再構成誤差を小さくする。

```math
\mathcal{L}_{rec}
=
\sum_t
\left(
x_t-\hat{x}_t
\right)^2
```

この設計の狙いは、global と local を別々の表現として持たせることで、系列全体の性質と局所的な変動を分けて扱うことである。

## 1.3 小売データへそのまま適用するときの懸念

小売データでは、売上の変動要因が複数ある。

| 要因 | 例 |
|---|---|
| 店舗・商品に固有の違い | ある店舗では特定の商品が普段からよく売れる |
| 日単位の違い | 休日、天気、販促により、その日全体の売上が変わる |
| 時間帯単位の違い | 夕方だけ売れやすい、朝だけ売れにくい |
| 日と時間帯の組み合わせ | 雨の日の夕方だけ通常と異なる |

元論文の global / local という 2 分割だけでは、小売データで重要な「日」と「時間帯」と「日×時間帯」を十分に分けて説明できない可能性がある。

また、Decoder が複数の表現をまとめて受け取る場合、予測精度が高くても、内部で情報が混ざる可能性がある。例えば、本来は時間帯成分として説明したい情報が、店舗・商品固有の表現に吸収されることがあり得る。

したがって、小売需要では単に global / local の潜在表現を作るだけでなく、出力される予測値をどの成分がどれだけ担当しているかを確認できる設計が必要になる。

# 2. FreshRetailNet-50K の適用の課題

## 2.1 データセットの特徴

FreshRetailNet-50K は、店舗・商品ごとの時間別売上を含む小売データである。

本研究では、店舗・商品系列を単位として、過去の売上や欠品、日次特徴量から将来の売上を扱う。

| 軸 | 内容 |
|---|---|
| 系列 | 店舗・商品 |
| 日 | 複数日の履歴と予測対象日 |
| 時間帯 | 1 日 24 時間 |
| 追加情報 | 値引き、休日、天気、欠品など |

このデータは、店舗・商品・日・時間帯という複数の軸を持つため、表現分離の題材としては適している。一方で、単純な基準値が非常に強いという特徴がある。

## 2.2 強い基準値の問題

FreshRetailNet-50K では、直近数日の同じ時間帯の平均が強い基準値になる。

例えば、ある商品の 18 時台の売上を予測する場合、直近数日の 18 時台の平均を使うだけで、かなり良い予測になる。

このとき、モデルが売上 $y$ を直接予測しても、次の問題が起きる。

| 問題 | 内容 |
|---|---|
| 基準値が強い | 複雑なモデルの改善幅が小さく見える |
| 残差が小さい | 学習対象が「残りかす」になりやすい |
| 解釈が難しい | 何をモデルが学習したのかが見えにくい |

そこで本研究では、売上そのものではなく、基準値で説明できなかった部分に注目する。

売上を $y_{i,d,h}$、基準値を $b_{i,d,h}$、残差を $r_{i,d,h}$ とすると、

```math
r_{i,d,h}
=
y_{i,d,h}
-
b_{i,d,h}
```

である。

残差にまだ日や時間帯の構造が残っていれば、その残差を分解することで、基準値では説明できなかった需要変動を読むことができる。

## 2.3 基準値によって残差の性質が変わる

これまでの実験から、FreshRetailNet-50K では基準値の選び方が重要であることが分かった。

| 基準値 | 意味 | 残差に残りやすい構造 | 結果の読み方 |
|---|---|---|---|
| `series_mean` | 系列ごとの平均を基準にする | 時間帯構造が残りやすい | 提案手法が効きやすい |
| `same_hour_recent_mean_d7` | 直近 7 日の同時刻平均を基準にする | 時間帯構造が吸収されやすい | 改善幅と解釈性が小さい |

この違いは、実験結果にも表れている。

| residual target | baseline MAE | corrected MAE | hour corr | 解釈 |
|---|---:|---:|---:|---|
| `series_mean_12k` | 0.0694 | 0.0487〜0.0494 | 0.9822〜0.9920 | 時間帯構造が残り、補正も効く |
| `same_hour_recent_mean_d7_12k` | 0.0574 | 0.0546〜0.0547 | -0.8861〜-0.8738 | 改善は小さく、時間帯成分としては読みにくい |

ここでの hour corr は、モデルが出した時間帯成分と、実際の残差の時間帯 profile がどの程度対応しているかを表す。

`series_mean` では hour corr が高く、時間帯成分として読める。一方、`same_hour_recent_mean_d7` では hour corr が負になり、時間帯成分が素直に残差の時間帯構造を説明しているとは言いにくい。



# 3. 提案手法

## 3.1 基本方針

提案手法の目的は、モデル内部の潜在表現を完全に識別することではない。

目的は、モデルが出力する残差予測を、意味のある成分に分けることである。

つまり、次の違いがある。

| 観点 | 潜在表現を分ける方針 | 出力成分を分ける方針 |
|---|---|---|
| 分ける対象 | モデル内部の表現 | モデルが出す残差予測 |
| 問題 | Decoder 内で混ざる可能性がある | 出力として成分を確認できる |
| 本研究の立場 | 補助的に使う | 主張の中心にする |

## 3.2 Encoder / Decoder の構成

ここで使う記号を先に整理する。

| 記号 | 意味 |
|---|---|
| $i$ | 店舗・商品系列の index |
| $d$ | 日の index |
| $h$ | 時間帯の index |
| $x_i$ | 系列 $i$ の入力特徴量。過去の売上、欠品、日次特徴量、時間特徴量などを含む |
| $y_{i,d,h}$ | 系列 $i$、日 $d$、時間帯 $h$ の売上 |
| $b_{i,d,h}$ | 基準値。例: 系列平均、直近同時刻平均 |
| $r_{i,d,h}$ | 基準値で説明できなかった残差。$r_{i,d,h}=y_{i,d,h}-b_{i,d,h}$ |
| $\hat r_{i,d,h}$ | モデルが予測する残差 |
| $z^g_i$ | 系列全体の表現 |
| $z^d_{i,d}$ | 日 $d$ に対応する表現 |
| $z^h_{i,h}$ | 時間帯 $h$ に対応する表現 |
| $z^u_{i,d,h}$ | 日 $d$ と時間帯 $h$ の組み合わせに対応する表現 |

入力 $x_i$ から、次の 4 種類の表現を作る。

```math
z^g_i
=
\mathrm{Enc}_g(x_i)
```

```math
z^d_{i,d}
=
\mathrm{Enc}_d(x_i, d)
```

```math
z^h_{i,h}
=
\mathrm{Enc}_h(x_i, h)
```

```math
z^u_{i,d,h}
=
\mathrm{Enc}_u(z^d_{i,d}, z^h_{i,h})
```

それぞれの意味は次の通りである。

| 表現 | 対応する成分 | 意味 |
|---|---|---|
| $z^g_i$ | 系列成分 | 店舗・商品に固有のズレ |
| $z^d_{i,d}$ | 日成分 | 日単位のズレ |
| $z^h_{i,h}$ | 時間帯成分 | 時間帯単位のズレ |
| $z^u_{i,d,h}$ | 日×時間帯成分 | 日と時間帯の組み合わせによるズレ |

重要なのは、これらを 1 つの Decoder にまとめて入れない点である。

各表現から、出力成分を別々に出す。

```math
\hat g_i = f_g(z^g_i)
```

```math
\hat a_{i,d} = f_d(z^d_{i,d})
```

```math
\hat c_{i,h} = f_h(z^h_{i,h})
```

```math
\hat u_{i,d,h} = f_u(z^u_{i,d,h})
```

最終的な残差予測は、それらの和である。

```math
\hat r_{i,d,h}
=
\hat g_i
+ \hat a_{i,d}
+ \hat c_{i,h}
+ \hat u_{i,d,h}
```

この設計により、予測残差をどの成分が担当しているかを後から確認できる。

## 3.3 残差分解の考え方

提案手法では、残差を次のように分解する。

```math
r_{i,d,h}
=
g_i
+ a_{i,d}
+ c_{i,h}
+ u_{i,d,h}
+ \varepsilon_{i,d,h}
```

これは、言葉で書くと次の意味である。

```text
基準値からのズレ
= 店舗・商品に固有のズレ
+ 日ごとのズレ
+ 時間帯ごとのズレ
+ 日と時間帯の組み合わせによるズレ
+ ノイズ
```

成分を分けることで、例えば次のような説明が可能になる。

| 見たいこと | 対応する成分 |
|---|---|
| この店舗・商品は普段から基準より外れやすいか | $g_i$ |
| ある日だけ全体的に売れたか | $a_{i,d}$ |
| 夕方だけ基準より売れたか | $c_{i,h}$ |
| 雨の日の夕方だけ特別に外れたか | $u_{i,d,h}$ |

## 3.4 成分の担当範囲と平均ゼロ制約

成分を複数用意するだけでは、意味のある分解にはならない。

例えば任意の定数 $\alpha$ について、

```math
g_i' = g_i + \alpha
```

```math
a_{i,d}' = a_{i,d} - \alpha
```

と置いても、

```math
g_i' + a_{i,d}'
=
g_i + a_{i,d}
```

である。つまり、予測値は変わらないが、どの成分が何を担当したかは変わってしまう。

そこで、各成分に平均ゼロ制約を置く。

日成分は、日方向の平均を 0 にする。

```math
\sum_d a_{i,d}=0
```

時間帯成分は、時間帯方向の平均を 0 にする。

```math
\sum_h c_{i,h}=0
```

日×時間帯成分は、日方向にも時間帯方向にも平均を持たないようにする。

```math
\sum_d u_{i,d,h}=0
\quad \forall h
```

```math
\sum_h u_{i,d,h}=0
\quad \forall d
```

この制約により、$u_{i,d,h}$ は「日だけの効果」や「時間帯だけの効果」を含まない。日と時間帯の組み合わせで初めて現れるズレだけを担当する。

観測された残差行列 $R_i \in \mathbb{R}^{D \times H}$ に対しては、概念的に次のような分解ができる。

```math
g_i = \bar r_i
```

```math
a_{i,d}
=
\bar r_{i,d,\cdot}
-
\bar r_i
```

```math
c_{i,h}
=
\bar r_{i,\cdot,h}
-
\bar r_i
```

```math
u_{i,d,h}
=
r_{i,d,h}
-g_i
-a_{i,d}
-c_{i,h}
```

ここで、

```math
\bar r_i
=
\frac{1}{DH}
\sum_{d,h}
r_{i,d,h}
```

である。

この形にすることで、残差を 1 つの不明な量として扱うのではなく、系列・日・時間帯・組み合わせのどこに由来するかを分けて扱える。

## 3.5 損失関数

主な損失は、残差再構成誤差である。

観測されているセルを $m_{i,d,h}=1$、観測されていないセルを $m_{i,d,h}=0$ とすると、再構成損失は次のように書ける。

```math
\mathcal{L}_{rec}
=
\frac{
\sum_{i,d,h}
m_{i,d,h}
\left|
r_{i,d,h}
-
\hat r_{i,d,h}
\right|
}{
\sum_{i,d,h}
m_{i,d,h}
}
```

さらに、成分が片側に寄り続けることを防ぐため、必要に応じて bias に関する制約を入れる。

概念的には、全体の損失は次の形である。

```math
\mathcal{L}
=
\mathcal{L}_{rec}
+
\lambda_{const}\mathcal{L}_{const}
+
\lambda_{bias}\mathcal{L}_{bias}
```

ここで、$\mathcal{L}_{const}$ は平均ゼロ制約、$\mathcal{L}_{bias}$ は補正後の予測が一方向にずれ続けることを抑える項である。

# 4. 現状の結果から言えること・言えないこと

## 4.1 実験の積み上げ

これまでの実験は、次の順に進めた。

| 段階 | 実験 | 目的 | 分かったこと |
|---|---|---|---|
| 残差診断 | 2-Exp-1 | 基準値 $b$ と残差 $r$ を確認 | 同時間帯系の基準値が強い |
| 初期モデル | 2-Exp-2〜7 | 残差再構成と swap 正則化 | 潜在分離だけでは安定しない |
| 合成データ(synthetic) | 2-Exp-8, 22 | 真の成分がある条件で検証 | 成分があれば回復できる |
| 出力分解 | 2-Exp-11〜15 | Decoder 出力を成分に分ける | 出力分解のほうがよかった |
| target 感度 | 2-Exp-16〜19 | FreshRetailNet で基準値を比較 | `series_mean` 誤差が主成功例 |
| 統計確認 | 2-Exp-20 | seed-level bootstrap | 改善が seed 依存ではない |
| 可視化 | 2-Exp-21 | hour profile / heatmap | 時間成分の対応を図示できる |
| 表の整理 | 2-Exp-23 | 論文用表を集約 | 本文表と appendix 表を分けられる |
| 規模確認 | 2-Exp-24 | 系列数を増やす | 12k でも `series_mean` は改善 |
| block 確認 | 2-Exp-25 | 系列開始位置を変える | 先頭系列だけに依存しない |
| 元論からの接続 | 2-Exp-26 | 通常予測で `global/local` と day/hour split を比較 | day/hour は小幅改善、interaction は不安定 |
| direct/residual bridge | 2-Exp-27 | `series_mean` residual で同じ latent split を比較 | residual は補正できるが、latent split だけでは `global/local` を上回らない |

## 4.2 合成データ(Synthetic) で言えること

synthetic では、真の成分 $g,a,c,u$ が分かっている。

そのため、モデルが出した成分が正しい成分に対応しているかを直接評価できる。

| 条件 | 結果 | 説明できること |
|---|---|---|
| base | global/day/hour はほぼ完全に回復、interaction も高く回復 | 成分が存在すれば提案手法は分解できる |
| no centering | 予測はできても成分相関が崩れる | 制約なしでは解釈性が崩れる |
| high interaction | interaction 成分が必要になる | 日×時間帯成分を持つ意味がある |
| high noise / small sample | 成分回復が落ちる | 適用条件と失敗条件を説明できる |

ここから言えるのは、提案手法が「必ずどんなデータでも分解できる」ということではない。

言えるのは、残差に成分構造があり、平均ゼロ制約が妥当な条件では、出力分解によって成分を回復できるということである。

## 4.3 FreshRetailNet で言えること

FreshRetailNet では真の成分は分からない。

そのため、synthetic のように「正解成分との相関」で評価することはできない。代わりに、次の観点で確認する。

| 観点 | 指標 | 意味 |
|---|---|---|
| 予測補正 | baseline MAE と corrected MAE | 基準値に残差補正を足して改善するか |
| 外れケース補正 | high residual top10 MAE | 基準値が外したケースを補正できるか |
| 時間帯成分 | hour corr | hour component が残差の時間帯 profile と対応するか |
| 成分寄与 | hour ablation delta | hour component を消すと悪化するか |
| bias | corrected bias | 補正が片側に寄っていないか |

主な結果は次の通りである。

| 実験 | 条件 | 結果 | 読み方 |
|---|---|---|---|
| 2-Exp-24 | `series_mean_12k` | baseline MAE 0.0694、corrected MAE 0.0487〜0.0494、hour corr 0.9822〜0.9920 | 系列数を増やしても改善と hour 対応が保たれる |
| 2-Exp-24 | `same_hour_recent_mean_d7_12k` | baseline MAE 0.0574、corrected MAE 0.0546〜0.0547、hour corr は負 | 強い基準値では残差の hour 構造が読みにくい |
| 2-Exp-25 | `series_mean_block0/1/2_6k` | MAE 改善幅 0.0173〜0.0190 | 先頭系列だけに依存しない |
| 2-Exp-25 | `same_hour_recent_mean_d7_block0/1/2_6k` | 改善幅は 0.0008〜0.0017 程度、hour corr は全て負 | 補正は小さく、成分解釈は弱い |
| 2-Exp-26 | direct forecasting | `global/day/hour` は `global/local` より WAPE を 0.6233 から 0.6133 に小幅改善、interaction 付きは WAPE 0.6267 | 通常予測では day/hour 分割は導入できるが、interaction までの強い根拠は弱い |
| 2-Exp-27 | `series_mean` residual latent split | `global/local` residual が corrected MAE 0.0593、day/hour は 0.0671、interaction 付きは 0.0614 | residual は学習可能だが、latent split だけでは不十分 |

## 4.4 なぜ残差を主対象にするか

2-Exp-26 では、元論文に近い `global + local` 分解を通常の future total forecasting に移植し、同じ反実仮想 global 正則化の下で、local を `day/hour/interaction` に分ける比較を行った。

結果として、`global + day + hour` は `global + local` より少し良かった。

| model | MAE | WAPE | bias |
|---|---:|---:|---:|
| `paper_global_local` | 1.4850 | 0.6233 | -0.4676 |
| `four_factor_global_day_hour` | 1.4611 | 0.6133 | -0.3787 |
| `four_factor_global_day_hour_interaction` | 1.4931 | 0.6267 | -0.2662 |

この結果から、local を day/hour に分けることは、小売需要に合った自然な拡張として導入できる。

しかし、通常予測では interaction まで入れても MAE/WAPE は改善しなかった。これは、売上全体 $y$ には系列水準、商品差、店舗差、曜日性、時間帯性、欠品などが同時に含まれ、day/hour/interaction の役割が混ざりやすいためである。

したがって、本研究では次のように問題を絞る。

```text
売上全体を直接 4 成分に分けるのではなく、
まず基準値 b で説明しやすい主効果を取り除き、
残った r = y - b に対して成分分解を行う。
```

この整理により、2-Exp-26 は「通常予測で 4 成分が常に勝つ」ことを示す実験ではなく、「元論文の global/local から day/hour 分割へ進む導入」として使う。

2-Exp-27 では、同じ `global/local` reference と day/hour split を `series_mean` residual に適用した。結果として、全モデルが baseline MAE 0.0721 を補正したが、最も良かったのは `global/local` residual で corrected MAE 0.0593 だった。day/hour は 0.0671、interaction 付きは 0.0614 であり、latent を細かく分けるだけでは `global/local` reference を上回らなかった。

この結果から、次のように整理する。

```text
残差には学習可能な構造が残っている。
しかし、潜在表現を細分化するだけでは、成分の役割が decoder 内で混ざり、安定した改善にはつながらない。
したがって、本研究の主張は latent split そのものではなく、
残差出力を global/day/hour/interaction に分け、centering 制約で意味を固定する出力分解に置く。
```

## 4.5 現時点で言えること

現時点で言えることは次である。

1. FreshRetailNet-50K では、基準値の選び方によって残差の性質が大きく変わる。
2. `series_mean` を基準値にすると、残差に時間帯構造が残り、提案手法による補正と hour 成分の解釈が成立しやすい。
3. `same_hour_recent_mean_d7` を基準値にすると、基準値が時間帯構造を吸収するため、残差の hour 成分は解釈しにくくなる。
4. `series_mean` residual の改善は、系列数を増やしても、系列ブロックを変えても保たれた。
5. synthetic では、真の成分がある条件で output decomposition が成分を回復できる。

6. 通常予測では day/hour 分割の効果は小幅であり、interaction 成分は不安定である。
7. residual target でも、latent split だけでは `global/local` reference を安定して上回らないため、出力分解と制約が必要である。

## 4.6 現時点で言えないこと

一方で、次のことはまだ言えない。

| 言えないこと | 理由 |
|---|---|
| FreshRetailNet-50K 全体で常に最良である | 全系列交差検証をしていないから |
| どんな基準値でも提案手法が改善する | 同時間帯の same-hour 系では改善幅が小さい |
| 実データで interaction 成分が強く効く | FreshRetailNet では hour 成分が主な成功例 |
| latent 表現そのものが完全に識別された | 保証しているのは出力成分の制約である |
| residual にすれば latent split だけで十分である | 2-Exp-27 では `global/local` residual が latent four-factor より良かった |


したがって、本研究の主張は
```text
提案手法は、強い基準値を置き換える手法ではなく、
基準値の後に残る残差に、再現可能な日・時間帯構造がある場合、
その残差を成分に分けて補正し、どの軸のズレなのかを説明する手法である。
```

## 4.7 応用上の意味

この分解ができると、単に予測値を出すだけでなく、次のような使い方ができる。

| 応用先 | 何が分かるか |
|---|---|
| 発注量の調整 | どの時間帯で基準より外れやすいか |
| 陳列・補充計画 | 夕方だけ需要が上がる商品を見つける |
| 店舗運営 | 店舗・商品に固有のズレを把握する |
| 異常検知 | 基準値から大きく外れるケースを検出する |
| 基準値診断 | same-hour baseline が強すぎて残差構造を消しているかを確認する |

特に、FreshRetailNet-50K のように単純な基準値が強いデータでは、予測精度だけを競うと手法の意味が見えにくい。

本研究の価値は、予測値そのものよりも、基準値から外れた部分をどの軸で説明できるかにある。

# Appendix A. 想定される質問と回答

## A.1 なぜ基準値からのズレを成分に分けると嬉しいのか

予測モデルが出す値を 1 つの数値として扱うだけでは、その予測が何によって変わったのかが分からない。

例えば、基準値より売上が高くなると予測されたとしても、それが

- その店舗・商品が普段から売れやすいからなのか
- その日だけ需要が高いからなのか
- 夕方など特定の時間帯だけ需要が高いからなのか
- ある日のある時間帯だけ特別に外れているからなのか

を区別できない。

そこで、残差予測を

```
\hat r_{i,d,h}
=
\hat g_i
+ \hat a_{i,d}
+ \hat c_{i,h}
+ \hat u_{i,d,h}
```

と分ける。

このとき嬉しい点は、予測補正を次のように読めることである。

| 成分 | 読み方 |
|---|---|
| $\hat g_i$ | この店舗・商品系列は、基準値から平均的にずれやすい |
| $\hat a_{i,d}$ | この日は、全体的に基準値からずれやすい |
| $\hat c_{i,h}$ | この時間帯は、基準値からずれやすい |
| $\hat u_{i,d,h}$ | この日と時間帯の組み合わせで特別にずれている |

つまり、この成分分けは「予測精度を上げるためだけの工夫」ではなく、「補正値を説明可能にするための工夫」である。

さらに重要なのは、売上 $y$ そのものではなく、基準値からのズレ $r=y-b$ を見ている点である。

売上を直接予測すると、モデルは「この商品は普段から売れる」「この時間帯は普段から売れる」といった、既に単純な基準値で説明できる大きな構造を再学習しやすい。

一方で、残差を見ると問いが変わる。

```text
売上はいくらか。
```

ではなく、

```text
既に使える基準値から、どこで、どの要因で外れるか。
```

を問うことになる。

このため、基準値 $b$ は「真の売上成分」を仮定しているものではない。既存の単純な予測、比較対象、診断軸として置いている。提案手法は $b$ を置き換えるのではなく、$b$ の後に残るズレを説明し、必要な場合に補正する。

## A.2 基準値が強いとはどういうことか

ここでいう基準値 $b$ は、モデルが学習する前に置く単純な予測である。

例えば `same_hour_recent_mean_d7` は、直近 7 日の同じ時間帯の平均を使う。

```
b_{i,d,h}
=
\frac{1}{7}
\sum_{k=1}^{7}
y_{i,d-k,h}
```

この基準値が強いとは、$b$ だけで $y$ の大部分を説明できるという意味である。

```
y_{i,d,h}
=
b_{i,d,h}
+
r_{i,d,h}
```

と書くと、基準値が強いほど、残差 $r$ は小さくなる。

ただし、これは「基準値という成分が真に存在する」と仮定しているというより、実務上よく効く説明済み部分を先に取り除いている、と考える方が自然である。

本研究の立場は次である。

```text
売上全体を一度に説明するのではなく、
単純な基準値で説明できる部分を先に除き、
残ったズレにどのような構造があるかを見る。
```

このため、基準値が強すぎる場合には、残差に時間帯構造がほとんど残らないことがある。FreshRetailNet の `same_hour_recent_mean_d7` はその例である。

## A.3 ブロック確認の「ブロック」とは何か

ここでいうブロックは、時間 window の分割ではない。

FreshRetailNet の系列を `store_id, product_id` で並べたときに、どの範囲の系列を使うかを変えたものである。

| block | 使う系列 |
|---|---|
| block0 | 先頭から 6000 系列 |
| block1 | 先頭 6000 系列を飛ばした次の 6000 系列 |
| block2 | 先頭 12000 系列を飛ばした次の 6000 系列 |

目的は、先頭付近の系列だけで偶然うまくいったのではないかを確認することである。

実験では、`series_mean` residual の改善が block0 / block1 / block2 のすべてで再現した。

したがって、少なくとも今回の範囲では、先頭系列だけに依存した結果ではないと説明できる。

## A.4 感度とは何か

ここでいう感度は、設定を少し変えたときに結論が変わるかどうかである。

例えば 2-Exp-24 では、使う系列数を変えた。

| 条件 | 目的 |
|---|---|
| 2000 系列 | 小さめの設定でも効果が出るか |
| 6000 系列 | 主実験規模で効果が出るか |
| 12000 系列 | 規模を増やしても効果が消えないか |

これを「系列数に対する感度確認」と呼んでいる。

2-Exp-25 では、系列の開始位置を変えた。これは「系列選択に対する感度確認」である。

感度確認の目的は、結論が特定の設定だけに依存していないことを確認することである。

## A.5 成分の担当範囲を固定するとは何か

ここで必要なのは、重なり合って説明されてしまう成分を、互いに役割が重ならないように分ける考え方である。

例えば残差を日成分と時間帯成分に分けたいとする。

もし制約がなければ、日成分に入れるべき平均的なズレを、時間帯成分に入れても予測値は同じになる。

つまり、

```
r = a + c
```

という和だけを見ていると、$a$ と $c$ の分け方は一意に決まらない。

そこで、

```
\sum_d a_{i,d}=0
```

```
\sum_h c_{i,h}=0
```

という制約を置く。

これにより、日成分は「日ごとの相対的なズレ」、時間帯成分は「時間帯ごとの相対的なズレ」として読めるようになる。

本文では、これを「直交分解」とは呼ばず、「成分の担当範囲を固定する」と説明する方がよい。

理由は、「直交分解」という語が、行列分解や厳密な線形代数的分解を連想させるためである。本研究で本文中に伝えたいことは、補正値を系列・日・時間帯・日×時間帯のどれが担当するかを曖昧にしない、という点である。

## A.6 平均ゼロ制約の本質

平均ゼロ制約の本質は、成分の担当範囲を決めることである。

例えば、日成分 $a_{i,d}$ に平均が残っていると、その平均部分は系列成分 $g_i$ と区別できない。

そこで、

```
\sum_d a_{i,d}=0
```

と置く。

これにより、日成分は「系列全体の平均的なズレ」ではなく、「ある日が他の日と比べてどれだけずれているか」を表す。

同様に、時間帯成分には

```
\sum_h c_{i,h}=0
```

を置く。

これにより、時間帯成分は「全体的に売れやすい」という情報ではなく、「どの時間帯が相対的に高いか低いか」を表す。

interaction 成分にはさらに強い制約を置く。

```
\sum_d u_{i,d,h}=0
\quad \forall h
```

```
\sum_h u_{i,d,h}=0
\quad \forall d
```

これにより、$u_{i,d,h}$ は日だけの効果でも時間帯だけの効果でもない、日と時間帯の組み合わせで初めて現れるズレを担当する。

# Appendix B. 損失関数の詳細

## B.1 残差再構成損失

主な損失は、観測された残差 $r_{i,d,h}$ と予測残差 $\hat r_{i,d,h}$ の差である。

観測されているセルを $m_{i,d,h}=1$、観測されていないセルを $m_{i,d,h}=0$ とすると、

```
\mathcal{L}_{rec}
=
\frac{
\sum_{i,d,h}
m_{i,d,h}
\left|
r_{i,d,h}
-
\hat r_{i,d,h}
\right|
}{
\sum_{i,d,h}
m_{i,d,h}
}
```

である。

今回の小売データでは欠品などにより観測できない売上があるため、$m_{i,d,h}$ によって観測セルだけを評価する。

## B.2 成分制約損失

出力成分が平均ゼロ制約を満たすように、次のような制約損失を入れる。

```
\mathcal{L}_{const}
=
\left\|
\sum_d \hat a_{i,d}
\right\|^2
+
\left\|
\sum_h \hat c_{i,h}
\right\|^2
+
\left\|
\sum_d \hat u_{i,d,h}
\right\|^2
+
\left\|
\sum_h \hat u_{i,d,h}
\right\|^2
```

この項は、成分の意味を保つための損失である。

## B.3 bias 制約

残差補正は、全体として片側に寄ることがある。

例えば、すべての予測を少し下げることで MAE が下がる場合があるが、その場合は補正後の予測が全体として過小になる。

そこで、補正後の bias を抑える。

```
\mathcal{L}_{bias}
=
\left|
\frac{1}{N}
\sum_{i,d,h}
\left(
y_{i,d,h}
-
(b_{i,d,h}+\hat r_{i,d,h})
\right)
\right|
```

概念的な全体損失は次の形である。

```
\mathcal{L}
=
\mathcal{L}_{rec}
+
\lambda_{const}
\mathcal{L}_{const}
+
\lambda_{bias}
\mathcal{L}_{bias}
```

実験では、MAE を重視する calibration と、bias を制約する calibration を比較した。結果として、bias 制約は全体 MAE ではやや不利になる場合があるが、高残差上位 10% の補正では有利になりやすかった。

## C.3 発表時に強調するべき結論

発表では、次の順に話すと分かりやすい。

1. 元論文は global / local を分ける研究である。
2. 小売では local を日・時間帯・日×時間帯に分けたい。

4. 残差に時間帯構造が残る target では、出力分解が補正と解釈に効く。
5. 強い same-hour baseline 後の残差では、構造が薄くなり、解釈性は下がる。
6. したがって、本研究は強い baseline を置き換えるのではなく、baseline 後の残差を説明可能に補正する研究である。

# Appendix D. データ利用の前提

今回の FreshRetailNet-50K 実験は、全店舗・全商品の全組み合わせに対する完全な交差検証ではない。

主な前提は次の通りである。

| 項目 | 今回の扱い |
|---|---|
| split | 公式の `train` / `eval` split を使用 |
| 系列単位 | `store_id, product_id` |
| 主実験の規模 | 最大 6000 train 系列、最大 1500 eval 系列 |
| 感度確認 | 2000 / 6000 / 12000 train 系列で確認 |
| ブロック確認 | 開始位置を変えた 3 つの 6000 系列ブロックで確認 |
| seed | 主に 3〜5 seed |

したがって、「FreshRetailNet-50K 全体で常に勝つ」とは主張しない。

一方で、追加実験により、`series_mean` の残差の傾向は、系列数を増やしても、系列ブロックを変えても保たれることを確認した。

# Appendix E. synthetic データの生成方法

## E.1 synthetic を使う理由

FreshRetailNet-50K では、真の「系列成分」「日成分」「時間帯成分」「日×時間帯成分」は観測できない。

そのため、実データだけでは、モデルが出した成分が本当に正しい成分に対応しているかを直接評価できない。

そこで synthetic データでは、あらかじめ真の成分を持つデータを作る。

$$
r_{i,d,h}
=
g_i
+ a_{i,d}
+ c_{i,h}
+ u_{i,d,h}
$$

このように真の $g,a,c,u$ を保存しておけば、モデルが出した

$$
\hat g_i,\quad
\hat a_{i,d},\quad
\hat c_{i,h},\quad
\hat u_{i,d,h}
$$

が正しい成分に対応しているかを、相関や MAE で評価できる。

## E.2 データ生成の全体像

synthetic では、売上を次のように生成する。

$$
y_{i,d,h}
=
b_{i,d,h}
+ r_{i,d,h}
+ \eta_{i,d,h}
$$

ここで、

| 記号 | 意味 |
|---|---|
| $i$ | 系列 |
| $d$ | 日 |
| $h$ | 時間帯 |
| $b_{i,d,h}$ | 基準となる売上パターン |
| $r_{i,d,h}$ | 真の残差成分の和 |
| $\eta_{i,d,h}$ | 観測ノイズ |

残差は、次の 4 成分から作る。

$$
r_{i,d,h}
=
g_i
+ a_{i,d}
+ c_{i,h}
+ u_{i,d,h}
$$

つまり synthetic は、提案手法が仮定している構造を持つように作る。

モデルに入力する特徴量には、過去の売上、欠品、値引き、休日、天気、時間特徴量、曜日特徴量、subgroup 情報を含める。

## E.3 基準値成分

まず、各系列 $i$ に基本的な売上水準を与える。

系列ごとに subgroup を 4 種類のうち 1 つ選ぶ。

$$
s_i \in \{0,1,2,3\}
$$

その subgroup に応じて、売れやすい時間帯の peak を変える。

```text
peak hour = 7, 12, 18, 21 のいずれか
```

基準値 $b_{i,d,h}$ は、系列ごとの基本水準と、時間帯に沿った周期パターンから作る。

概念的には次の形である。

$$
b_{i,d,h}
=
\mathrm{base}_i
+
\mathrm{amp}_i
\left(
1+
\cos
\frac{2\pi(h-\mathrm{peak}_i)}{H}
\right)
$$

ここで、$H=24$ である。

この基準値は、売上全体にある大きな時間帯パターンを表す。

## E.4 global 成分

global 成分 $g_i$ は、系列 $i$ に固有のズレである。

実装では、subgroup と乱数から系列ごとの定数として作る。

$$
g_i
=
\alpha_g
\left(
\xi_i
+
0.25(s_i-1.5)
\right)
$$

ここで、

| 記号 | 意味 |
|---|---|
| $\alpha_g$ | global 成分の強さ |
| $\xi_i$ | 系列ごとの乱数 |
| $s_i$ | subgroup |

である。

2-Exp-22 の base 条件では、`global_scale = 0.55` を使った。

## E.5 day 成分

day 成分 $a_{i,d}$ は、曜日、休日、値引き、天気から作る。

実装上は、曜日ごとの基本効果を置き、さらに値引きや休日の効果を足す。

概念的には次の形である。

$$
\tilde a_{i,d}
=
\alpha_d
\left(
\mathrm{weekday}(d)
+ 1.5 \cdot \mathrm{discount}_{i,d}
+ 0.25 \cdot \mathrm{holiday}_{d}
- 0.08 \cdot \max(\mathrm{weather}_{i,d},0)
\right)
$$

ただし、このままだと day 成分に系列全体の平均が残るため、平均を引く。

$$
a_{i,d}
=
\tilde a_{i,d}
-
\frac{1}{D}
\sum_{d'}
\tilde a_{i,d'}
$$

これにより、day 成分は「日ごとの相対的なズレ」になる。

2-Exp-22 の base 条件では、`day_scale = 0.7` を使った。

## E.6 hour 成分

hour 成分 $c_{i,h}$ は、subgroup ごとに異なる時間帯パターンとして作る。

具体的には、sin 波と peak 周辺の山を組み合わせる。

$$
\tilde c_{i,h}
=
\sin
\frac{2\pi(h-\mathrm{peak}_i)}{H}
+
0.65
\exp
\left(
-
\frac{(h-\mathrm{peak}_i)^2}{2 \cdot 2.5^2}
\right)
$$

その後、時間帯方向の平均を引く。

$$
c_{i,h}
=
\alpha_h
\left(
\tilde c_{i,h}
-
\frac{1}{H}
\sum_{h'}
\tilde c_{i,h'}
\right)
$$

これにより、hour 成分は「どの時間帯が相対的に高いか低いか」を表す。

2-Exp-22 の base 条件では、`hour_scale = 0.8` を使った。

## E.7 interaction 成分

interaction 成分 $u_{i,d,h}$ は、値引きや休日がある日に、特定の時間帯だけ追加で変化する効果として作る。

まず、13 時付近に山を持つ promotion shape を作る。

$$
\mathrm{promo}(h)
=
\exp
\left(
-
\frac{(h-13)^2}{2 \cdot 2.2^2}
\right)
$$

この shape から平均を引き、時間帯方向の平均を 0 にする。

そして、値引きと休日によって強さを変える。

$$
\tilde u_{i,d,h}
=
\alpha_u
\left(
2.8 \cdot \mathrm{discount}_{i,d}
+ 0.35 \cdot \mathrm{holiday}_d
\right)
\mathrm{promo}(h)
$$

最後に、日方向と時間帯方向の平均を引く。

$$
u_{i,d,h}
\leftarrow
u_{i,d,h}
-
\frac{1}{D}
\sum_{d'}
u_{i,d',h}
$$

$$
u_{i,d,h}
\leftarrow
u_{i,d,h}
-
\frac{1}{H}
\sum_{h'}
u_{i,d,h'}
$$

これにより、$u_{i,d,h}$ は日だけの効果でも時間帯だけの効果でもない、日と時間帯の組み合わせで初めて現れるズレを担当する。

2-Exp-22 の base 条件では、`interaction_scale = 0.6` を使った。

## E.8 ノイズと欠品

売上には観測ノイズを足す。

$$
y_{i,d,h}
=
\max
\left(
b_{i,d,h}
+ r_{i,d,h}
+ \eta_{i,d,h},
0.05
\right)
$$

base 条件では、`noise_std = 0.12` を使った。

また、小売データに近づけるため、欠品も発生させる。

欠品確率は基本の `stockout_rate` に、売上が高い時間帯ほど少し上がる項を加える。

```text
stockout_rate = 0.03
```

欠品したセルでは、観測売上を 0〜25% 程度に落とし、mask で観測できないセルとして扱う。

## E.9 2-Exp-22 の base 設定

2-Exp-22 の base 条件は次の通りである。

| 項目 | 値 |
|---|---:|
| 系列数 | 1500 |
| 日数 | 35 |
| 時間帯数 | 24 |
| 予測対象日数 | 2 |
| recent days | 7 |
| noise std | 0.12 |
| stockout rate | 0.03 |
| global scale | 0.55 |
| day scale | 0.7 |
| hour scale | 0.8 |
| interaction scale | 0.6 |
| valid ratio | 0.15 |
| test ratio | 0.15 |

## E.10 難易度シナリオ

2-Exp-22 では、base 条件に加えて次のシナリオを作った。

| scenario | 変更点 | 目的 |
|---|---|---|
| `low_interaction` | `interaction_scale = 0.2` | interaction が弱いとき、interaction 成分の寄与が小さくなるか |
| `high_interaction` | `interaction_scale = 1.2` | interaction が強いとき、interaction 成分が必要になるか |
| `high_noise` | `noise_std = 0.35` | ノイズが大きいと成分回復が崩れるか |
| `short_history` | `days = 21` | 観測日数が短いと分解が不安定になるか |
| `small_sample` | `n_series = 500` | 系列数が少ないと成分回復が落ちるか |
| `high_stockout` | `stockout_rate = 0.15` | 欠品が多いと残差学習が崩れるか |
| `low_hour_signal` | `hour_scale = 0.2` | hour signal が弱いと hour 成分の寄与が小さくなるか |

## E.11 synthetic で評価したこと

synthetic では真の成分が分かっているため、次を直接評価できる。

| 指標 | 意味 |
|---|---|
| global corr | $\hat g$ が真の $g$ に対応しているか |
| day corr | $\hat a$ が真の $a$ に対応しているか |
| hour corr | $\hat c$ が真の $c$ に対応しているか |
| interaction corr | $\hat u$ が真の $u$ に対応しているか |
| total component MAE | 成分の和が真の残差に近いか |
| ablation delta | 成分を消したときに性能がどれだけ悪化するか |

主な結果は次の通りである。

| scenario | global corr | day corr | hour corr | interaction corr | 読み方 |
|---|---:|---:|---:|---:|---|
| `base` | 0.9995 | 0.9979 | 0.9991 | 0.9654 | 標準条件では成分を回復できる |
| `high_interaction` | 0.9994 | 0.9982 | 0.9990 | 0.9890 | interaction が強い条件で interaction 成分が必要 |
| `small_sample` | 0.9912 | 0.9801 | 0.9902 | 0.4252 | サンプル数が少ないと interaction 回復が崩れる |
| `high_noise` | 0.9990 | 0.9795 | 0.9948 | 0.8334 | ノイズが大きいと成分回復が弱くなる |

この synthetic 実験により、提案手法は「成分が存在する条件では回復できる」ことと、「ノイズやサンプル数によって失敗し得る」ことを説明できる。
