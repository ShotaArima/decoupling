# summary

# 1. 元論文の問題意識
- 元論文である「decoupling Local and Global Representations of Time Series」は、時系列データから得られる潜在表現を異なる性質の2種類に分けて扱うことを目的としている

| 表現 | 意味 | 小売データでの例 |
| --- | --- | --- |
|  global | 1系列全体に通じる性質 | ある店舗・商品が普段どの程度売れるのか |
| local | 時点・区間ごとの変化 | ある日、ある時間帯で売上げが変化すること |

- 研究では、これら2つの潜在表現を学習し、下流タスク(予測など)への適用を提案していた

## モデル構成
- Encoder
    - 入力された時系列から2種類の表現を作成する 
        - $ \boldsymbol{z}_g$ は系列全体に通じる表現
        - $\boldsymbol{z}_{l,t}$ 時点または短い区間に対応する表現

```math
\begin{align*}z_g&=\mathrm{Enc}_g(\boldsymbol{x}_{1;T})\\ z_l&=\mathrm{Enc}_l(\boldsymbol{x}_{t-w;t})\end{align*}
```

- Decoder
    - 2つの潜在表現を入力とし、時系列を再構成する

```math
\hat{x}_t = \mathrm{Dec}(\boldsymbol{z}_g, \boldsymbol{z}_{l,t})
```

- 学習では、再構成誤差を小さくする

```math
\mathcal{L}_{\mathrm{rec}} = \|x_t-\hat{x}_t\|^2
```
## 自分の問題意識
- 小売では、売上げが様々な要因によって変動する
    - その商品がよく売れる商品である
    - その日だけの需要が高かった
    - 夕方などの特定の時間帯の売上げが増えた
- このような需要を予測する際に、何の要因によって生じたのかを理解することが重要

## そのまま適用するときの懸念  <--ここは要修正
- 元論文での実験データは病気のメンタルモデル <--ここは要修正
- しかし、小売データでは、誰か特定の人をモデル内部の表現で直接観測できない
- 予測精度が高い場合でも、内部では情報が混ざっている可能性があり、時間帯の情報として表現されるべきものが、系列固有の情報として表現されることもあり得る  <--ここは要修正

# 2. データセットFreshRetailNet-50Kについて
- 店舗・商品ごとの時間別売上げを含む小売データ
- 日毎の情報に加え、1日の中の時間帯別売上げが含まれるため、店舗・商品・日・時間という複数軸で需要を分析することができる

## このデータセット特有の特徴
- 直近同時刻平均が強いデータセットである
    - 複雑なモデルによる予測よりも、単純な基準値強い場面が多くあった
    - ある商品の18時台の売上げを予測するにあたって、直近数日の18時台の売上げを基準にする方法が精度がよかった

# FreshRetailNetへの適用について
- FreshRetailNet-50Kは、直近の同時刻平均が強いことから、それでは説明できない部分にかんする表現学習を行うことにした
    - 売上げを $y$ 、基準値を $b$ 、残差を $r$ としたときに以下の関係になる $$r = y-b$$
    - この残差に日付や時間帯の構造が残っていれば、その需要の構造や変化などを追うことができるのではと思っています

| 基準 | 基準値だけのMAE | 補正後MAE |
| --- | --- | --- | 
| `series_mean_all` |  0.0697 | 0.0501  | 
| `same_hour_recent_mean_d7_all` | 0.0580 | 0.0565 |

- 以上より、FreshRetailNet-50K における課題は、単にモデルを⾼性能にすることではなく、どのような基準値を置き、その後にどのような残差を学習対象にするかにあると分かった。


# 3. 提案手法

##  提案の基本方針

今回の提案は、モデル内部の潜在表現を完全に分離することではなく、モデルが出力する残差予測を、意味のある成分に分けることを目標にする。

## 提案手法の Encoder / Decoder 構成

提案手法でも、入力から複数の表現を作る点は元論文と同じである。

ただし、Decoder の設計を変える。入力から、次の 4 種類の表現を作る。

```math
z^g_i = \mathrm{Enc}_g(x_i)
```

```math
z^d_{i,d} = \mathrm{Enc}_d(x_i, d)
```

```math
z^h_{i,h} = \mathrm{Enc}_h(x_i, h)
```

```math
z^u_{i,d,h} = \mathrm{Enc}_u(z^d_{i,d}, z^h_{i,h})
```

ここで、$z^g_i$ は系列成分、$z^d_{i,d}$ は日成分、$z^h_{i,h}$ は時間帯成分、$z^u_{i,d,h}$ は日と時間帯の組み合わせ成分に対応する。

重要なのは、これらを 1 つの Decoder にまとめて入れるのではなく、出力成分を別々に出す点である。

$$
\hat g_i = f_g(z^g_i)
$$

$$
\hat a_{i,d} = f_d(z^d_{i,d})
$$

$$
\hat c_{i,h} = f_h(z^h_{i,h})
$$

$$
\hat u_{i,d,h} = f_u(z^u_{i,d,h})
$$

最終的な残差予測は、それらを足し合わせる。

```math
\hat r_{i,d,h} = 
\hat g_i+
 \hat a_{i,d}+
 \hat c_{i,h}+
 \hat u_{i,d,h}
```

元論文に近い構成では、Decoder が

```math
\hat r_{i,d,h}=\mathrm{Dec}(z^g_i,z^d_{i,d},z^h_{i,h},z^u_{i,d,h})
```

のように一括で予測する。これに対して、提案手法では出力を明示的に分解する。

この違いを表にすると、次のようになる。

| 項目 | 元論文に近い構成 | 今回の提案 |
|---|---|---|
| Encoder | global / local 表現を作る | 系列 / 日 / 時間帯 / 日×時間帯の表現を作る |
| Decoder | 表現をまとめて入力し、1 つの予測を出す | 各表現から成分を別々に出す |
| 出力 | $\hat r = \mathrm{Dec}(z)$ | $\hat r=\hat g+\hat a+\hat c+\hat u$ |
| 解釈 | Decoder 内で混ざる可能性がある | 出力成分として確認できる |

##  残差を 4 つの成分に分ける

提案手法では、残差を次の 4 つの成分に分けて考える。

| 成分 | 意味 | 小売データでの例 |
|---|---|---|
| 系列成分 | 店舗・商品に固有のズレ | この商品は基準よりやや売れやすい |
| 日成分 | 日単位のズレ | 休日や天気により、その日全体で売上が変わる |
| 時間帯成分 | 時間帯単位のズレ | 夕方だけ基準より売れやすい |
| 日×時間帯成分 | 日と時間帯の組み合わせによるズレ | 特定の日の夕方だけ通常と異なる |

数式を使わずに表せば、提案手法は次の分解を行う。

```text
基準値からのズレ
= 系列に固有のズレ
+ 日ごとのズレ
+ 時間帯ごとのズレ
+ 日と時間帯の組み合わせによるズレ
+ ノイズ
```

この分解により、単に売上を予測するだけでなく、予測補正がどの要因によって生じたのかを確認できる。

数式で書くと、仮定している残差構造は次の形である。

```math
r_{i,d,h}
=
g_i
+ a_{i,d}
+ c_{i,h}
+ u_{i,d,h}
+ \varepsilon_{i,d,h}
```

ここで、$g_i$ は系列成分、$a_{i,d}$ は日成分、$c_{i,h}$ は時間帯成分、$u_{i,d,h}$ は日と時間帯の組み合わせ成分である。

この式が意味するのは、残差を 1 つの不明な量として扱うのではなく、どの軸のズレなのかを分けて扱うということである。

## 直交分解と制約
- 成分を複数用意するだけでは、意味のある分解にはならない。
- 例 : 任意の定数 $\alpha$ について、

$$
g_i' = g_i + \alpha
$$

$$
a_{i,d}' = a_{i,d} - \alpha
$$

と置いたとしても以下のように $\alpha$ がうめられてしまう

$$
g_i' + a_{i,d}' = g_i + a_{i,d}
$$

このままでは、系列成分と日成分のどちらが何を担当するかが決まらない。そこで、各成分に次の制約を置く。日成分は、日方向の平均を 0 にする。

$$
\sum_d a_{i,d}=0
$$

時間帯成分は、時間帯方向の平均を 0 にする。

$$
\sum_h c_{i,h}=0
$$

日×時間帯成分は、日方向にも時間帯方向にも平均を持たないようにする。

$$
\sum_d u_{i,d,h}=0
\quad \forall h
$$

$$
\sum_h u_{i,d,h}=0
\quad \forall d
$$

この制約により、$u_{i,d,h}$ は「日だけの効果」や「時間帯だけの効果」を含まない。日と時間帯の組み合わせで初めて現れるズレだけを担当する。

このような分解は、観測された残差行列 $R_i \in \mathbb{R}^{D \times H}$ に対して、次のように構成できる。

$$
g_i = \bar r_i
$$

$$
a_{i,d} = \bar r_{i,d,\cdot}-\bar r_i
$$

$$
c_{i,h} = \bar r_{i,\cdot,h}-\bar r_i
$$

```math
u_{i,d,h}
=
r_{i,d,h}
-g_i
-a_{i,d}
-c_{i,h}
```

ここで、

```math
\bar r_i
=
\frac{1}{DH}\sum_{d,h}r_{i,d,h}
```

```math
\bar r_{i,d,\cdot}
=
\frac{1}{H}\sum_h r_{i,d,h}
```

```math
\bar r_{i,\cdot,h}
=
\frac{1}{D}\sum_d r_{i,d,h}
```

である。

この形にすると、各成分の担当範囲が決まる。したがって、予測値だけでなく、予測値を構成する成分の意味も検証しやすくなる。

## 損失関数の設計

提案手法では、主な損失は残差の再構成誤差である。

観測されているセルを $m_{i,d,h}=1$、観測されていないセルを $m_{i,d,h}=0$ とすると、再構成損失は次のように書ける。

```math
\mathcal{L}_{rec}
=
\frac{
\sum_{i,d,h}
m_{i,d,h}
\left|
r_{i,d,h}-\hat r_{i,d,h}
\right|
}{
\sum_{i,d,h}m_{i,d,h}
}
```

さらに、出力成分が制約を満たすように、制約損失を加える。

```math
\mathcal{L}_{const}
=
\left\|\sum_d \hat a_{i,d}\right\|^2
+
\left\|\sum_h \hat c_{i,h}\right\|^2
+
\left\|\sum_d \hat u_{i,d,h}\right\|^2
+
\left\|\sum_h \hat u_{i,d,h}\right\|^2
```

最終的な損失は、概念的には次の形である。

```math
\mathcal{L}
=
\mathcal{L}_{rec}
+
\lambda_{const}\mathcal{L}_{const}
+
\lambda_{bias}\mathcal{L}_{bias}
```

$\mathcal{L}_{bias}$ は、補正後の予測が全体として一方向にずれ続けることを抑えるための項である。

今回の FreshRetailNet-50K 実験では、予測誤差だけでなく、補正後の bias と高残差ケースの改善も重要であった。そのため、単に平均誤差を下げるだけでなく、補正が過剰に片側へ寄らないようにする設計も検討した。
