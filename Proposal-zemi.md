# 1. decoupling global and local data の論文について

## 1.1 元論文の問題意識

元論文 *Decoupling Local and Global Representations of Time Series* は、時系列データから得られる表現を、性質の異なる 2 種類の情報に分けて扱うことを目的としている。

ここでいう `global` と `local` は、データセット全体と個別サンプルを単純に分けるという意味ではない。より正確には、1 つの時系列系列に通底する比較的変わりにくい性質と、時点・区間ごとに変化する性質を分けるという考え方である。

小売データに置き換えると、次のように理解できる。

| 表現 | 意味 | 小売データでの例 |
|---|---|---|
| global | 系列に通底する性質 | ある店舗・商品が普段どの程度売れるか |
| local | 時点・区間ごとの変化 | ある日、ある時間帯だけ売上が変化すること |

たとえば、ある商品の売上が高い場合でも、その理由は 1 つではない。

- その商品がもともとよく売れる商品である。
- その日だけ需要が高かった。
- 夕方など特定の時間帯だけ売上が増えた。
- 欠品や値引きなどにより、通常と異なる動きが生じた。

元論文は、このような要因が 1 つの表現に混ざることを避け、系列に通底する情報と局所的な変化を分けて学習しようとした研究である。

## 1.2 この考え方が有用な理由

時系列予測では、最終的な予測値が当たることが重要である。しかし、研究としては予測値だけでなく、その予測がどのような要因によって生じたのかを理解したい。

特に小売需要では、次のような問いが重要になる。

- 店舗・商品そのものの違いが効いているのか。
- 曜日や休日など、日単位の要因が効いているのか。
- 朝、昼、夕方など、時間帯の要因が効いているのか。
- 特定の日と時間帯の組み合わせが効いているのか。

元論文の global/local 分離は、こうした問いに対して、表現を分けることで答えようとする枠組みである。

## 1.3 そのまま適用したときの懸念

一方で、元論文の枠組みには、今回の小売データに適用する上で注意すべき点がある。

元論文では、主にモデル内部の表現を分ける。つまり、モデルの中で `global` と `local` に相当する潜在表現を持たせ、それらを用いて予測や再構成を行う。

しかし、モデル内部の表現は直接観測できない。したがって、モデルが `global` と `local` を持っていたとしても、その中身が本当に人間の想定する意味に対応しているとは限らない。

予測精度が高い場合でも、内部では情報が混ざっている可能性がある。たとえば、本来は時間帯の情報として表現されるべきものが、系列固有の情報として表現されることもあり得る。

この点が、FreshRetailNet-50K に適用した際の重要な課題になった。

# 2. FreshRetailNet-50K の適用の課題

## 2.1 FreshRetailNet-50K の特徴

FreshRetailNet-50K は、店舗・商品ごとの時間別売上を含む小売データである。日ごとの情報に加えて、1 日の中の時間帯別売上が含まれるため、店舗・商品、日、時間帯という複数の軸で需要を分析できる。

このため、一見すると global/local 分離の考え方とは相性がよい。店舗・商品ごとの違い、日ごとの違い、時間帯ごとの違いを分けて扱える可能性があるためである。

しかし、実際に適用すると、単純な表現分離だけでは十分でないことが分かった。

## 2.2 直近同時刻平均が非常に強い

FreshRetailNet-50K では、複雑なモデルよりも、単純な基準値が強い場面が多かった。

特に強かったのは、直近の同じ時間帯の平均を使う方法である。たとえば、ある商品の 18 時台の売上を予測する際に、直近数日の 18 時台の売上を基準にする方法である。

小売では、これは自然な結果である。商品ごとの時間帯パターンは比較的安定しており、同じ時間帯の直近の動きが将来の売上を強く説明するためである。

この結果は、FreshRetailNet-50K において、単純な同時刻パターンが非常に大きな説明力を持つことを示している。

## 2.3 強い基準値の後には学習すべき構造が残りにくい

今回の研究では、売上そのものではなく、基準値では説明できなかった部分に注目した。

売上を `y`、基準値を `b`、残差を `r` とすると、次のように表せる。

```text
r = y - b
```

この残差 `r` にまだ曜日や時間帯の構造が残っていれば、モデルがそれを学習する意味がある。

しかし、基準値 `b` が強すぎる場合、残差 `r` には学習すべき構造がほとんど残らない。直近同時刻平均は、時間帯に関する規則性をあらかじめ多く吸収してしまう。その結果、モデルが追加で学ぶ余地は小さくなる。

実験でも、`same_hour_recent_mean` 系の基準値を用いた場合、残差に残る時間帯構造は弱く、提案モデルによる改善も小さかった。

## 2.4 基準値の選び方で残差の性質が変わる

一方、系列ごとの平均を基準値にした場合は、時間帯ごとの構造が残差に残りやすかった。

この条件では、提案モデルによる補正が有効であった。

| 条件 | 基準値だけの MAE | 補正後 MAE | 解釈 |
|---|---:|---:|---|
| `series_mean_all` | 0.0697 | 0.0501 | 改善が大きい |
| `same_hour_recent_mean_d7_all` | 0.0580 | 0.0565 | 改善は小さい |

この結果から、FreshRetailNet-50K における課題は、単にモデルを高性能にすることではなく、どのような基準値を置き、その後にどのような残差を学習対象にするかにあると分かった。

## 2.5 FreshRetailNet-50K で見えた問題点

これまでの結果から、FreshRetailNet-50K に元論文の考え方を適用する際の問題点は、次のように整理できる。

| 問題点 | 内容 |
|---|---|
| 基準値が強い | 直近同時刻平均だけで売上の多くが説明される |
| 残差構造が基準値に依存する | 基準値が強いほど、残差に学習すべき構造が残りにくい |
| 内部表現の意味が不安定 | `global` や `local` と呼んでも、本当に意味が分かれているとは限らない |
| 実データの相互作用は見えにくい | 日と時間帯の組み合わせ効果は、合成データほど明確には確認しにくい |

したがって、元論文の方法をそのまま適用するのではなく、FreshRetailNet-50K に合わせて、残差をどのように定義し、どのように分解するかを再設計する必要がある。

# 3. 提案手法

## 3.1 提案の基本方針

今回の提案は、モデル内部の潜在表現を完全に分離することを目標にしない。

代わりに、モデルが出力する残差予測を、意味のある成分に分けることを目標にする。

これは、次のような発想の転換である。

```text
モデル内部の表現が完全に分かれていることを保証する
```

のではなく、

```text
出力された残差が、検証可能な形で分かれていることを保証する
```

ことを目指す。

この方針により、予測結果を後から成分ごとに確認しやすくなる。

## 3.2 残差を 4 つの成分に分ける

提案手法では、残差を次の 4 つの成分に分けて考える。

| 成分 | 意味 | 小売データでの例 |
|---|---|---|
| 系列成分 | 店舗・商品に固有のズレ | この商品は基準よりやや売れやすい |
| 日成分 | 日単位のズレ | 休日や天気により、その日全体で売上が変わる |
| 時間帯成分 | 時間帯単位のズレ | 夕方だけ基準より売れやすい |
| 日×時間帯成分 | 日と時間帯の組み合わせによるズレ | 特定の日の夕方だけ通常と異なる |

数式を使わずに表せば、提案手法は次の分解を行う。

```text
基準値からのズレ
= 系列に固有のズレ
+ 日ごとのズレ
+ 時間帯ごとのズレ
+ 日と時間帯の組み合わせによるズレ
+ ノイズ
```

この分解により、単に売上を予測するだけでなく、予測補正がどの要因によって生じたのかを確認できる。

## 3.3 出力に制約を入れる理由

成分を複数用意するだけでは、意味のある分解にはならない。

たとえば、本来は時間帯成分が表すべき情報を、系列成分が吸収してしまう可能性がある。その場合、予測値はある程度当たっていても、成分の解釈は崩れる。

そこで、提案手法では、各成分が担当する範囲を制約する。

直感的には、次のような制約である。

- 日成分は、日ごとの差を表すが、全体の平均的な水準は持たない。
- 時間帯成分は、時間帯ごとの差を表すが、全体の平均的な水準は持たない。
- 日×時間帯成分は、日だけの効果や時間帯だけの効果を含まない。

このように、各成分が持ってよい情報を限定することで、成分の意味を後から検証しやすくする。

## 3.4 FreshRetailNet-50K で期待する効果

FreshRetailNet-50K では、基準値の後に残る構造が条件によって異なる。

そのため、提案手法が常に大きく改善するとは限らない。むしろ、提案手法が有効なのは、残差にまだ日や時間帯の構造が残っている場合である。

実験では、`series_mean_all` の条件で時間帯構造が残り、時間帯成分が実際の残差パターンと強く対応した。

この条件では、提案手法により次の効果が確認できた。

- 全体の誤差が下がった。
- 誤差が大きいケースでも改善した。
- 時間帯成分が何を説明しているかを確認できた。

一方、`same_hour_recent_mean` 系の条件では、時間帯構造の多くが基準値によって説明済みであり、提案手法による追加改善は小さかった。

したがって、提案手法の位置づけは次のようになる。

```text
残差に構造が残っている場合に、その構造を補正し、解釈するための方法
```

であり、

```text
あらゆる基準値を常に上回る汎用的な予測器
```

ではない。

# 4. 現状の結果から言えること言えないこと

## 4.1 言えること: 合成データでは成分分解が成立する

真の成分が分かる合成データでは、提案手法は成分を高い精度で回復した。

代表的な結果は次の通りである。

| 成分 | 真の成分との一致 |
|---|---:|
| 系列成分 | 0.9995 |
| 日成分 | 0.9979 |
| 時間帯成分 | 0.9991 |
| 日×時間帯成分 | 0.9654 |

この結果から、成分が実際に存在する条件では、提案した分解が有効に働くことが分かった。

また、制約を外したモデルでは成分の解釈が崩れた。したがって、成分を分けるためには、単に出力の箱を分けるだけでなく、各成分が担当する範囲を制約する必要がある。

## 4.2 言えること: FreshRetailNet-50K では時間帯構造が残る条件で効果がある

`series_mean_all` の条件では、基準値だけの場合よりも、提案手法による補正後の方が良い結果になった。

| 指標 | 基準値だけ | 補正後 |
|---|---:|---:|
| 全体 MAE | 0.0697 | 0.0501 |
| 高残差上位 10% | 0.2788 | 0.1914 |

さらに、時間帯成分は実際の残差の時間帯パターンと強く対応していた。

このことから、FreshRetailNet-50K においても、残差に時間帯構造が残っている条件では、提案手法が予測補正と解釈の両面で有効であると言える。

## 4.3 言えること: 強い同時刻基準値の後では改善は小さい

`same_hour_recent_mean_d7_all` では、補正による改善は小さかった。

| 指標 | 基準値だけ | 補正後 |
|---|---:|---:|
| 全体 MAE | 0.0580 | 0.0565 |

これは、提案手法が無効であることを意味しない。むしろ、同時刻基準値が時間帯構造をすでに多く説明しているため、残差側に追加で学習すべき構造があまり残っていないことを示している。

したがって、この結果は提案手法の失敗というより、適用条件を示す結果として解釈するのが妥当である。

## 4.4 まだ言えないこと: FreshRetailNet-50K 全体で常に有効とは言えない

現時点の結果から、提案手法が FreshRetailNet-50K の全ての条件で常に最良であるとは言えない。

言えるのは、残差に時間帯構造が残る条件では効果があるということである。

一方、強い同時刻基準値の後では改善が小さいため、提案手法は基準値の設計と切り離して評価できない。

## 4.5 まだ言えないこと: 実データで日×時間帯成分を強く主張するには不足がある

合成データでは、日×時間帯成分も高い精度で回復できた。

しかし、FreshRetailNet-50K では、日×時間帯成分が強く効いていることを示す結果はまだ十分ではない。

したがって、現時点では、実データにおける主な成功例は時間帯成分である。日×時間帯成分については、合成データで成立条件を示し、実データでは今後の検証課題として扱うのが適切である。

## 4.6 まだ言えないこと: 内部表現が完全に分離したとは言えない

今回の提案は、モデル内部の潜在表現が完全に識別されたことを主張するものではない。

主張できるのは、モデルが出力する残差成分について、意味を持つように制約し、後から検証できる形にしたという点である。

したがって、本研究の主張は次のように整理する。

```text
内部表現の完全な分離を保証するのではなく、
出力された残差を検証可能な成分として分ける。
```

## 4.7 ゼミで伝えたい要点

本研究の流れは、次のように整理できる。

```text
元論文は、時系列の系列に通底する性質と局所的な変化を分けることを目指した。

FreshRetailNet-50K に適用すると、直近同時刻平均が非常に強く、
その後の残差には学習すべき構造が残りにくい場合があることが分かった。

そこで、売上そのものではなく、基準値からの残差に注目した。
さらに、その残差を系列成分、日成分、時間帯成分、日×時間帯成分に分ける方法を提案した。

この方法は、残差に時間帯構造が残る条件では有効であった。
一方で、同時刻基準値が時間帯構造をほぼ説明している条件では、改善は小さかった。

したがって、提案手法は万能な予測器ではなく、
残差に残る構造を補正し、解釈するための方法である。
```
